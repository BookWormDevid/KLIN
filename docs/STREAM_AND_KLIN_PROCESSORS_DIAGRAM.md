## 15. Important Notes

- The current Litestar router exposes offline KLIN endpoints, but does not currently expose active stream start/stop/status endpoints. `StreamService` exists and is testable, but `api_router` only wires `KlinController`.
- Offline inference is gated by a quick X3D pass. If X3D predicts no violence, `InferenceProcessor.analyze()` returns immediately and does not run MAE or YOLO.
- Streaming inference is also gated, but dynamically. X3D acts as a lightweight trigger that enables the heavier MAE and YOLO stages for a cooldown window.
- Read DTO mapping now lives in `app/application/mappers/klin.py`, not inside the DTO classes themselves.
- `KlinService` no longer imports `app_settings` directly; retry configuration is injected through `IKlinRuntimeSettings`.
- `StreamService.stop_stream()` emits `STREAM_STOPPED` through the same event producer and queue, but `StreamEventConsumer.handle()` currently only supports `YOLO`, `MAE`, and `X3D_VIOLENCE`.
- `KlinResultDto` stores stage outputs as JSON strings, while the streaming path moves structured `StreamEventDto` payloads and persists them into dedicated stream event tables.
